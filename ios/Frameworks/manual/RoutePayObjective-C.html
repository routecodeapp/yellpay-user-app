Payment (Objective-C)

About the Environment

This SDK provides a site setup for environments other than production. (※ Currently, only the production environment is available.)
If the method has an argument called EnvironmentMode environmentMode, you can select the server environment. The configuration values are as follows:

Environment

URL

Production

EnvironmentModeEnumProduction

RoutePay Class

Name

Description

Class Name

RoutePay

Overview

Handles class initialization, credit card registration, and payment processing.

info.plist

If you use schema startup, you must set up a custom URL scheme in info.plist.

  • Example

  <key>CFBundleURLTypes</key> 
  <array> 
    <dict> 
      <!-- identifier --> 
      <key>CFBundleURLName</key> 
      <string>routepay</string> 
      <!-- URL Scheme --> 
      <key>CFBundleURLSchemes</key> 
      <array> 
        <string>routepay</string> 
      </array> 
      </dict> 
  </array>

  • CFBundleURLSchemes

    • If you set the scheme parameter in the server’s resist token issuing API, it must match the value of the scheme parameter.

  • Host part

    • The value of the host part is unidauth.platfield.com.

    • Please route it with the scheme in AppDelegate.

Member Registration + Identity Verification (Without Environment Mode)

This function initiates member registration and identity verification processes.

The “User Status Retrieval” API determines the following status changes based on conditions:

Status Code

Description

-1

Errors other than identity verification errors

1

Initialization (Before Member Registration)

2

Member Registration Completed

3

Applying (Pending Application)

4

Accepted (Awaiting Verification) or Processing (Under Verification)

5

CompletedError (Verification Complete with Errors)

6

Completed (Verification Complete)

Workflow

  1. Initialization (Before Member Registration):
    Execute “Member Information Registration” via WebView (with script callback capabilities).

  2. Member Registration Completed:
    Retrieve the eKYC URL through the Identity Verification URL Retrieval API.

  3. Applying (Pending Application):
    Launch “Member Information Registration” using the Safari module on iOS.

  4. Accepted (Awaiting Verification) or Processing (Under Verification):
    Launch “Awaiting Identity Verification” screen via WebView (with script callback capabilities).

  5. CompletedError (Verification Complete with Errors):
    Launch “Identity Verification Error” screen via WebView (with script callback capabilities).

  6. Completed (Verification Complete):
    Launch “Identity Verification Completed” screen via WebView (with script callback capabilities).
