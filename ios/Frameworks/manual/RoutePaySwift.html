決済(Swift)

環境について

メソッドの引数に「EnvironmentMode environmentMode」ある場合、 サーバー接続先として、本番環境または、検証環境を選択出来ます。 設定が可能な値は、以下の通りです。

また、環境ごとにサービスIDの発行が必要です。 契約時に接続する環境をご確認ください。

環境

EnvironmentMode

本番

EnvironmentModeEnum.production

検証

EnvironmentModeEnum.staging

名称

内容

クラス名

RoutePaySwift

概要

クラスの初期化、クレジットカード登録および決済を行います。

info.plist

If you use schema startup, you must set up a custom URL scheme in info.plist.

  • Example

  <key>CFBundleURLTypes</key> 
  <array> 
    <dict> 
      <!-- identifier --> 
      <key>CFBundleURLName</key> 
      <string>routepay</string> 
      <!-- URL Scheme --> 
      <key>CFBundleURLSchemes</key> 
      <array> 
        <string>routepay</string> 
      </array> 
      </dict> 
  </array>

  • CFBundleURLSchemes

    • If you set the scheme parameter in the server’s resist token issuing API, it must match the value of the scheme parameter.

  • Host part

    • The value of the host part is unidauth.platfield.com.

    • Please route it with the scheme in AppDelegate.

会員登録 + 本人確認 (環境モードあり)

会員登録および本人確認起動処理

「ユーザー状態取得」APIを取得して、 条件により

ステータス

説明

-1

本人確認検証エラー以外のエラー

1

初期化 (会員登録前)

2

会員登録終了

3

applying(申請中)

4

accepted(確認待ち) or processing(確認中)

5

completedError(確認完了:エラー)

6

completed(確認完了)

が変更されるので

1.初期化 (会員登録前)

WebView(スクリプトコールバック可能)にて、 「会員情報登録」を実行

2.会員登録終了

API 本人確認URL取得でekyc URLを取得する

3.applying(申請中)

iOSでは、Safariモジュールにて 「会員情報登録」を起動

4.accepted(確認待ち) or processing(確認中) WebView(スクリプトコールバック可能)にて、 「本人確認待ち画面」を起動

5.completedError(確認完了:エラー)
WebView(スクリプトコールバック可能)にて、 「本人確認エラー画面」を起動

6.completed(確認完了)
WebView(スクリプトコールバック可能)にて、 「本人確認完了画面」を起動
